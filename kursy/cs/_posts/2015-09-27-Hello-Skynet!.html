Tworzymy nasz pierwszy projekt.

Zakładam, że już zainstalowałeś Visual Studio Community, a jeżeli nie, to jest to ostatni moment, żeby to zrobić. Gdy już się uporasz z tym procesem, pora uruchomić nasze IDE, napisać swój pierwszy program oraz poznać kilka elementów C#.

  1. Wybierz File > New > Project lub naciśnij Ctrl + Shift + N.
  2. Z listy rozwijanej wybierz Visual C#, a następnie Console Application.
  3. U dołu okna w polu Name wpisz "Skynet".
  4. W polu Location wybierz folder, w którym chcesz przechowywać programy z tego kursu.
  5. Upenij się że checkbox Create directory for solution jest zaznaczone.
  6. Kliklamy Ok.

Po chwili otworzy sie nam okno edytora, gdzie wpisujemy (Tak, wpisujemy, od kopiuj-wklej jeszcze nikt niczego się nie nauczył.) kod naszego pierwszego programu :

{% highlight csharp %} using static System.Console; namespace Skynet { class Program { static void Main(string[] args) { string name; WriteLine("Witaj, jak masz na imię?"); name = ReadLine(); WriteLine("Miło mi Cię poznać " + name + ", jestem Skynet."); } } } {% endhighlight %}

Teraz, aby skompilować i uruchomić program, który będzie odpowiedzialny za zniszczenie ludzkości, wystarczy nacisnąć Ctrl + F5. Dzięki tej kombinacji uruchomimy naszą aplikację poza środowiskiem Visual Studio, dzięki czemu zobaczymy jak będzie się zachowywać po normalnym uruchomieniu, czyli w tzw. Środowisku Proukcyjnym. Jeżeli chcemy śledzić błędy w naszej aplikacji uruchomimy ją w Trybie Debugowania przyciskiem F5.

Czas na analizę

Dyrektywy

Jak pewnie zwróciliście uwagę, z kilku using zostało wam tylko jedno, a do tego zmodyfikowane. Słowo to jest Dyrektywą kompilatora. Są to wiadomości zostawiane kompilatorowi przez programistę, które informują go o tym, co musi "pamiętać" podczas kompilacji. Użycie naszej pozwala nam na wywoływanie metod oraz używania typów znajdującyć się w przestrzeni nazw, którą napisaliśmy po tym słowie. Gdybyśmy nie użyli tej dyrektywy wywołanie metody WriteLine wyglądałoby tak :

{% highlight csharp %} System.Console.WriteLine("Witaj, jak masz na imię?"); {% endhighlight %}

Jeżeli kompilator byłby człowiekiem moglibyśmy mu powiedzieć "Hej, kolego. Jeżeli przy wywołaniu metody nie zobaczysz jej przestrzeni nazw poszukaj jej w System.Console". Część z was pewnie zwróciła uwagę, że nie opisałem dlaczego zostało użyte słówko static, opisze to nieco niżej.

Elementy Programowania Obiektowego

Z racji tego że C# jest językiem w pełni obiektowym oraz wymusza na nas Programowanie Zorientowane Obiektowo czuję konieczność "muśniecia" tego tematu, poprzez krótkie opisanie użytych przez nas elementów tego podejścia.

Przestrzeń nazw

Pod naszą dykrektywą pojawia się kolejne słowo kluczowe namespace. Jest to wspomniana już wyżej przestrzeń nazw. U nas nosi ona nazwę "Skynet". W językach pokroju naszego C# jest to swoisty pojemnik, który jest elementem organizacyjnym programu. Może zawierać inne przestrzenie nazw oraz klasy. Aby uzyskać dostęp do elementu podrzędnego względem Przestrzeni Nazw należy użyć kropki, która jest tzw. akcesorem. W naszym programie została ona użyta w tej roli raz. Zgadniecie gdzie?

Klasy

Klasa jest kolenym elementem organizacji programu. Stanowi ona podobnie jak przestrzeń nazw zbiornik zawierający głównie metody i właściwości, ale co ważniejsze jest ona wzorem, który pozwala tworzyć nam egzemplarze danej klasy zwane obiektami, na rzecz których możemy używać metod klasy, z której pochodzą, po to by modyfikować ich właściwości. Jednak o tym nieco później. U nas tą klasą jest "Program".

Metody

Są to wykonywalne fragmenty klas, których użycie prowadzi do zmiany stanu obiektu, na którym została wywołana. Teraz rozłożmy nieco naszego main'a, który jest naszą pierwszą oraz najważniejszą metodą, lecz najpierw przypomnijmy sobie jak wyglądał :

{% highlight csharp %} static void Main(string[] args) { ... } {% endhighlight %}

Zatem do rzeczy :

  1. static - użycie go tutaj skutkuje tym, że możemy użyć tej metody bez tworzenia obiektu klasy "Program". Jest ono wymagane przy metodzie Main.
  2. void - Każda metoda musi być poprzedzona nazwą typu danych, dzięki czemu komputer będzie wiedział jakiego typu będą dane, które będzie musiał obsłużyć po jej wykonaniu. Jest to jedno z wymagań języka "typo-bezpiecznego". Typ void(ang. pusty) oznacza, że nasza metoda nie generuje żadnych danych.
  3. Main - Nazwa naszej specjalnej metody będąca zarazem słowem kluczowym. Nazwy pozwalają nam zidentyfikować metodę, co umożliwia łatwe jej wywołanie. Dodatkowo main jest nazwą zastrzeżoną i stanowi główą metodę w programie. Jest to miejsce z której nasz komputer uruchamia program.
  4. (string[] args) - Po nazwie znajdziemy jejParametry. Służą one do przekazywania do jej ciała danych, na których ma pracować. Podczas określania właściwości parametru nalezy podać jego typ, w naszym wypadku jest to string (a dokładnie tablica stringów), oraz nazwę, której będziemy uzywać w metodzie,by odowałać się do danych
  5. { ... } - nawiasy klamrowe służą do określenia granic metody.

Ciało metody

Uff, po długiej podróży udało nam się dotrzeć do samego serca naszego programu, czyli do ciała metody main, które wyglądało tak :

{% highlight csharp %} string name; WriteLine("Witaj, jak masz na imię?"); name = ReadLine(); WriteLine("Miło mi Cię poznać " + name + ", jestem Skynet."); {% endhighlight %}

Teraz zobaczymy co się stało w poszczególych linijkach :

  1. Dokonujemy definicji zmiennej o nazwie "name" typu string. W ten sposób mówimy kompilatorowi, że Skynet będzie używał zmiennej o indetyfikatorze "name" oraz żeby zarezerował na nią ilośc pamięci stosowną do typu string, który umożliwai przechowywanie ciągów znaków, zamniętych w podwójnym cudzysłowiu.
  2. Wywołujemy WriteLine, która wyświetla w konsoli tekst podany jako jej parametr. Jest to podstawowy sposób przekazywania informacji człowiekowi przez komputer.
  3. Teraz Skynet za pomocą metody ReadLine oraz operatora przypisania (=) zapamięta nasze imię w zmiennej "name".
  4. W ostatnim kroku przywita się z nami znowu wyświetlając nam tekst, jednak zanim to zrobi dokona konkatynacji, czyli dodania do siebie dwóch fragmentów zdania, oraz zmiennej "name" za pomocą operatora +

Warto zwrócić uwagę, że wywołanie metod ReadLine, oraz WriteLine nie byłoby możliwe, gdyby nie były to metody statyczne, oraz że każda operacja w ciele musi być oddzielona średnikiem (ang. semicolon)
